DLP Testing Types:
Incoming DLP: Tests detection of threats coming INTO your organization (can use any sender)
Outgoing DLP: Tests prevention of data leaving your organization (requires organizational sender)
Incoming DLP Testing:
• Tests your organization's ability to detect threats in incoming emails
• Can use AWS SES sender (external threat simulation)
• Tests: Malware detection, sensitive data recognition, phishing identification
Outgoing DLP Testing:
• Tests your organization's ability to prevent data leakage
• REQUIRES organizational SMTP sender credentials
• Tests: Data loss prevention policies, sensitive data blocking, compliance enforcement
Outgoing DLP Requirements:
• Must use valid organizational email credentials
• SMTP authentication must be enabled for your tenant
• Contact IT if you get "SmtpClientAuthentication is disabled" errors
Must be from your organization's domain
Recommended: Generate app password for security
Verify credentials before sending
